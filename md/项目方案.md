## 一、项目目标与范围

### 1.1 老师 PPT 核心要求

- **从"检测"走向"解释"**：不仅给恶意/良性标签，还要解释"为什么"
- **大数据 → 少样本（few-shot）推理**：用少量代表性样本做高效检测
- **单一模型 → 知识增强**：结合安全规则库，让 LLM "有据可依"

### 1.2 任务范围（精简）

| 任务 | 是否做 | 理由 |
|------|--------|------|
| **Task A** 恶意/良性二分类（必做） | ✅ | PPT 必做项 |
| **Task D** 行为解释 | ✅ | RAG 的天然产物 |
| **消融实验 1/2/3** | ✅ | PPT 必做 |
| **消融实验 4**（Core-set） | ✅ | 加分项，代码简单 |
| Task B/C 多分类/多标签 | ❌ | Drebin 没有家族标签 |
| Task E 规则改进 | ❌ | 时间允许再加 |

### 1.3 不做的事

- ❌ 不训练神经网络（标题"NN OR LLM"二选一，走 LLM 路线）
- ❌ 不解析 APK 文件（Drebin 已是处理好的 CSV）
- ❌ 不用 LangChain（直接 FAISS + Ollama 更清晰）
- ❌ 不微调（Fine-tune）Gemma

## 二、数据集（Drebin-215）说明

### 2.1 数据集来源

- **学术来源**：Daniel Arp et al., *"Drebin: Effective and Explainable Detection of Android Malware in Your Pocket"*, NDSS 2014
- **本地路径**：`data/Android Malware Dataset for Machine Learning/drebin-215-dataset-5560malware-9476-benign.csv`
- **特征类别说明**：`data/Android Malware Dataset for Machine Learning/dataset-features-categories.csv`

### 2.2 数据规模与平衡性

| 指标 | 值 |
|------|---|
| 总样本数 | **15,036** |
| 良性样本（B = Benign） | 9,476（63%） |
| 恶意样本（S = Spam/Malware） | 5,560（37%） |
| 特征数 | 215 + 1 标签列 |
| 数据格式 | 0/1 二值矩阵 |

### 2.3 特征类别分布

| 类别 | 数量 | 例子 |

| **Manifest Permission**（权限） | 113 | `SEND_SMS`, `READ_PHONE_STATE`, `INSTALL_PACKAGES` |
| **API call signature**（API 调用） | 73 | `transact`, `bindService`, `Ljava.lang.Class.getMethods` |
| **Intent**（系统意图） | 23 | `android.intent.action.BOOT_COMPLETED` |
| **Commands signature**（系统命令） | 6 | `mount`, `system/bin/su` |

> 💡 **关键洞察**：特征已经按 PPT 要求的"Permission / API / Network" 三类预分好类了，我们的规则库直接按这个分类设计即可。

### Step 4 — K-shot 采样（`02_kshot_sampling.ipynb`）

```python
def random_kshot(df, k=5, seed=42):
    """每类随机选 K 个样本作为参考"""
    benign = df[df['class'] == 'B'].sample(k, random_state=seed)
    malware = df[df['class'] == 'S'].sample(k, random_state=seed)
    return pd.concat([benign, malware]).reset_index(drop=True)

def coreset_kshot(df, label_col='class', k=5, seed=42):
    """每类用 K-Means 选 K 个代表性样本"""
    from sklearn.cluster import KMeans
    selected = []
    for label in df[label_col].unique():
        sub = df[df[label_col] == label].reset_index(drop=True)
        X = sub.drop(columns=[label_col]).values
        km = KMeans(n_clusters=k, random_state=seed, n_init=10).fit(X)
        for c in km.cluster_centers_:
            idx = ((X - c) ** 2).sum(axis=1).argmin()
            selected.append(sub.iloc[idx])
    return pd.DataFrame(selected).reset_index(drop=True)

def stratified_testset(df, n_per_class=100, exclude_idx=None, seed=42):
    """从剩余样本平衡采样测试集"""
    if exclude_idx is not None:
        df = df.drop(index=exclude_idx)
    pieces = [df[df['class'] == c].sample(n_per_class, random_state=seed)
              for c in df['class'].unique()]
    return pd.concat(pieces).sample(frac=1, random_state=seed).reset_index(drop=True)
```

### Step 6 — 实现 Ollama 调用封装（`src/ollama_client.py`）

```python
import ollama
import json

def ollama_generate(prompt, model='gemma4:e4b', system=None):
    messages = []
    if system:
        messages.append({'role': 'system', 'content': system})
    messages.append({'role': 'user', 'content': prompt})

    resp = ollama.chat(
        model=model,
        messages=messages,
        format='json',                       # 关键：强制 JSON
        options={'temperature': 0.1},
    )
    return resp['message']['content']

def parse_json(text):
    try:
        return json.loads(text)
    except json.JSONDecodeError:
        # 用正则兜底
        import re
        m = re.search(r'\{.*\}', text, re.DOTALL)
        if m:
            try:
                return json.loads(m.group())
            except:
                return None
        return None
```

### Step 7 — 样本特征转自然语言（`src/prompt_builder.py`）

```python
def sample_to_text(row, cats_df):
    """把 Drebin 一行特征转换为可读描述"""
    feature_col_to_cat = dict(zip(cats_df['feature'], cats_df['category']))
    activated = {
        'Permission': [],
        'API': [],
        'Intent': [],
        'Commands': [],
    }
    for col, val in row.items():
        if col == 'class' or val != 1:
            continue
        cat = feature_col_to_cat.get(col, 'Other')
        if 'Permission' in cat:
            activated['Permission'].append(col)
        elif 'API' in cat:
            activated['API'].append(col)
        elif 'Intent' in cat:
            activated['Intent'].append(col)
        elif 'Commands' in cat:
            activated['Commands'].append(col)

    parts = []
    if activated['Permission']:
        parts.append(f"Permissions: {', '.join(activated['Permission'][:15])}")
    if activated['API']:
        parts.append(f"API calls: {', '.join(activated['API'][:15])}")
    if activated['Intent']:
        parts.append(f"Intents: {', '.join(activated['Intent'][:10])}")
    if activated['Commands']:
        parts.append(f"Commands: {', '.join(activated['Commands'])}")
    return ' | '.join(parts)
```

### Step 8 — RAG Pipeline（`04_rag_pipeline.ipynb`）

```python
def retrieve_rules(query, embedder, index, rules_df, top_k=3):
    q_emb = embedder.encode([query], normalize_embeddings=True).astype('float32')
    _, ids = index.search(q_emb, top_k)
    return rules_df.iloc[ids[0]]

def rag_predict(sample_row, cats_df, embedder, index, rules_df, system_prompt):
    sample_text = sample_to_text(sample_row, cats_df)
    retrieved = retrieve_rules(sample_text, embedder, index, rules_df, top_k=3)
    prompt = build_prompt(sample_text, retrieved)
    raw = ollama_generate(prompt, system=system_prompt)
    return parse_json(raw), raw
```

### Step 9 — 消融实验（`05_ablation_study.ipynb`）

每个实验跑 200 个测试样本，结果保存到 `results/predictions/exp{N}.jsonl`：

```python
from tqdm import tqdm

def run_experiment(test_df, mode, cats_df, embedder=None, index=None,
                   rules_df=None, system_prompt=None):
    results = []
    for idx, row in tqdm(test_df.iterrows(), total=len(test_df), desc=mode):
        sample_text = sample_to_text(row, cats_df)

        if mode == 'baseline':                     # Exp 1
            prompt = build_baseline_prompt(sample_text)
        elif mode == 'full_rules':                 # Exp 2
            prompt = build_full_rules_prompt(sample_text, rules_df)
        elif mode == 'rag':                        # Exp 3 / Exp 4
            retrieved = retrieve_rules(sample_text, embedder, index,
                                       rules_df, top_k=3)
            prompt = build_rag_prompt(sample_text, retrieved)

        raw = ollama_generate(prompt, system=system_prompt)
        parsed = parse_json(raw)

        results.append({
            'idx': int(idx),
            'true_label': row['class'],
            'parsed': parsed,
            'raw': raw,
        })
    return results
```

## 八、关键代码骨架

### 8.1 极简的一次 RAG 推理

```python
import ollama, json, pandas as pd
from sentence_transformers import SentenceTransformer
import faiss

# 一次性加载
embedder = SentenceTransformer('BAAI/bge-small-en-v1.5')
rules_df = pd.read_csv('rules/rules.csv')
index = faiss.read_index('rules/rules.index')
cats_df = pd.read_csv('data/Android Malware Dataset for Machine Learning/'
                      'dataset-features-categories.csv',
                      header=None, names=['feature', 'category'])

def predict_one(sample_row):
    # 1. 样本特征转文本
    sample_text = sample_to_text(sample_row, cats_df)

    # 2. RAG 检索
    q_emb = embedder.encode([sample_text], normalize_embeddings=True).astype('float32')
    _, ids = index.search(q_emb, 3)
    retrieved = rules_df.iloc[ids[0]]

    # 3. 拼 prompt
    rules_text = "\n".join(
        f"- [{r['rule_id']}] {r['feature_pattern']} => "
        f"{r['related_label']}: {r['explanation']}"
        for _, r in retrieved.iterrows()
    )
    prompt = f"""## Sample Features
{sample_text}

## Retrieved Security Rules
{rules_text}

## Task
Decide whether this Android app is Malware ('S') or Benign ('B').
Output JSON only with keys: label, evidence, rules_cited, explanation, confidence."""

    # 4. Ollama 推理
    resp = ollama.chat(
        model='gemma4:e4b',
        messages=[
            {'role': 'system', 'content': 'You are an Android malware analyst.'},
            {'role': 'user', 'content': prompt},
        ],
        format='json',
        options={'temperature': 0.1},
    )
    return json.loads(resp['message']['content'])
```

### 9.3 规则库设计（`rules/rules.csv`）— 20 条核心规则

格式：`rule_id, feature_pattern, related_label, category, explanation`

**Permission 类规则（8 条）**

| rule_id | feature_pattern | related_label | category | explanation |
|---------|----------------|---------------|----------|-------------|
| R001 | SEND_SMS + RECEIVE_SMS + INTERNET | S | Permission | 同时具备短信收发和联网能力，符合短信木马模式 |
| R002 | READ_SMS + WRITE_SMS + READ_CONTACTS | S | Permission | 读写短信+读取联系人，典型信息窃取组合 |
| R003 | INSTALL_PACKAGES + DELETE_PACKAGES + SYSTEM_ALERT_WINDOW | S | Permission | 静默安装/卸载+悬浮窗，可下载安装其他恶意应用 |
| R004 | RECORD_AUDIO + ACCESS_FINE_LOCATION + INTERNET | S | Permission | 录音+精确位置+联网，典型间谍软件特征 |
| R005 | WRITE_APN_SETTINGS + MODIFY_PHONE_STATE | S | Permission | 篡改网络配置和电话状态，极少有正常用途 |
| R006 | RECEIVE_BOOT_COMPLETED + SYSTEM_ALERT_WINDOW + WAKE_LOCK | S | Permission | 开机自启+悬浮窗+保持唤醒，持久化特征明显 |
| R007 | INTERNET + ACCESS_NETWORK_STATE（仅这些） | B | Permission | 只申请联网相关基础权限，绝大多数普通应用模式 |

|---------|----------------|---------------|----------|-------------|
| R009 | SmsManager + TelephonyManager.getDeviceId + getSubscriberId | S | API | 短信操作+设备指纹收集，常见信息上传模式 |
| R010 | DexClassLoader + System.loadLibrary + Runtime.getRuntime | S | API | 动态加载代码+本地库+命令执行，恶意 payload 解密执行特征 |
| R011 | Ljavax.crypto.Cipher + Ljavax.crypto.spec.SecretKeySpec | S | API | 加密 API 重度使用，可能用于隐藏 C&C 通信 |
| R012 | HttpGet.init + DefaultHttpClient.execute | S/B | API | 网络通信，看具体地址和内容才能判断 |
| R013 | bindService + onServiceConnected + transact | B | API | 标准服务绑定模式，多数正规应用都用 |
| R014 | TelephonyManager.getLine1Number + getSubscriberId + getSimSerialNumber | S | API | 大量收集 SIM/号码标识，典型设备指纹收集 |

**Intent 类规则（4 条）**

| rule_id | feature_pattern | related_label | category | explanation |
|---------|----------------|---------------|----------|-------------|
| R015 | android.intent.action.BOOT_COMPLETED + RECEIVE_BOOT_COMPLETED 权限 | S/B | Intent | 开机自启，看配合什么权限：配合敏感权限多为恶意 |
| R016 | android.provider.Telephony.SMS_RECEIVED + READ_SMS 权限 | S | Intent | 监听短信接收（非短信应用），高度可疑 |
| R017 | android.intent.action.PACKAGE_ADDED + INSTALL_PACKAGES 权限 | S | Intent | 监听应用安装+具备安装权限，典型 dropper 恶意软件 |
| R018 | android.intent.action.MAIN + android.intent.category.LAUNCHER（仅这些） | B | Intent | 标准应用入口配置，绝大多数应用都有 |

**Commands 类规则（2 条）**

| rule_id | feature_pattern | related_label | category | explanation |
|---------|----------------|---------------|----------|-------------|
| R019 | mount + system/bin/su | S | Commands | 尝试挂载文件系统+使用 su，典型 root 提权特征 |
| R020 | （没有任何 Commands 特征） | B | Commands | 不调用底层系统命令，多数正常应用都不需要 |

## 十一、常见坑与避雷指南

### 11.1 数据相关

- 🚨 **class 列是字符串 'S'/'B'**：不是 0/1，注意对比
- 🚨 **少数特征名带特殊字符**：如 `Ljava.lang.Class.getMethods`，prompt 里要原样保留
- 🚨 **测试集要类别平衡**：从全集随机抽 → 大概率 63% 良性，导致 accuracy 虚高

### 11.2 Ollama 相关

- 🚨 **端口冲突**：默认 11434，被占用用 `OLLAMA_HOST=0.0.0.0:11435 ollama serve`
- 🚨 **GPU 没识别**：Windows 需要 NVIDIA 驱动+CUDA；`ollama ps` 看是否走 GPU
- 🚨 **CPU 模式慢**：200 样本可能要 30 分钟，先用 50 个试跑

### 11.3 Python SDK 相关

- 🚨 **temperature 别设 0**：Ollama 设 0 会触发 bug，用 0.1
- 🚨 **format='json' 偶尔失败**：用正则兜底解析
- 🚨 **首次推理慢**：模型加载需要 5-10 秒，第二次开始快

### 11.4 RAG 相关

- 🚨 **embedding 维度对齐**：规则库和查询用**同一个** embedder
- 🚨 **Top-K=3 即可**：你笔记第二讲讲过 Lost in the middle，K 太大反而差
