# 少样本实验改造实现方案

## 0. 已确认决策

本实现严格按以下决策落地：

1. `K` 表示每类抽取 `K` 个训练/示例样本，因此训练集总数是 `2K`。
2. 测试/验证集采用每类固定数量抽样，默认每类 `100` 个。
3. 不同 `K` 使用同一个测试集；实现时先抽固定测试集，再从测试集之外抽训练集。
4. 保存到 CSV 的标签只使用原始语义标签 `B` / `S`，不额外保存 `label_id`。
5. 第一版只实现单个 `seed` 参数，默认 `42`。
6. 不预留 Core-set / 多策略接口，因此不加 `--strategy`，只做 random few-shot。

## 1. 本轮代码改造范围

新增 4 个根目录脚本/模块：

```text
fewshot_utils.py
prepare_fewshot_split.py
baseline_fewshot.py
llm_only_fewshot.py
```

保留现有 `baseline-train.ipynb` 作为 80/20 传统 ML 上限参考，不在主实验中使用它做公平对比。`test.py` 暂不修改。

本轮不实现 RAG、Core-set、多 seed、模型微调，也不修改原始 `data/` 文件。

## 3. `prepare_fewshot_split.py`

### 3.1 命令

```bash
python prepare_fewshot_split.py --k 5 --seed 42 --test-per-class 100
```

可选参数：

```text
--data-path              默认 data/drebin-215-dataset-5560malware-9476-benign.csv
--output-root            默认 data/processed
--label-col              默认 class
```

不添加 `--strategy`，不添加多 seed 参数。

### 3.2 抽样规则

读取原始 CSV 后：

1. 规范化 `class` 列，只允许 `B` / `S`。
2. 先从全量数据中每类抽 `test_per_class` 个样本，得到固定 `test.csv`。
3. 从 `test.csv` 之外的数据中，每类抽 `K` 个样本，得到 `train.csv`。
4. 训练集和测试集必须无重叠。
5. 保存的 `train.csv` 和 `test.csv` 保持原始 215 个特征列 + `class` 列，`class` 只写 `B` / `S`。

为了让 K 曲线更稳定，同一个 `seed` 下训练样本采用“每类固定随机顺序，取前 K 个”的方式。这样 `K=1` 的训练样本会包含在 `K=3` 中，`K=3` 会包含在 `K=5` 中。

### 3.3 元数据

`test_metadata.json` 保存：

```json
{
  "seed": 42,
  "test_per_class": 100,
  "label_col": "class",
  "label_format": "B/S",
  "test_size": {"B": 100, "S": 100, "total": 200},
  "test_sha256": "...",
  "test_source_indices": [1, 2, 3]
}
```

`k5/metadata.json` 保存：

```json
{
  "k": 5,
  "k_semantics": "per_class",
  "seed": 42,
  "test_per_class": 100,
  "train_size": {"B": 5, "S": 5, "total": 10},
  "test_size": {"B": 100, "S": 100, "total": 200},
  "test_sha256": "...",
  "train_source_indices": [1, 2, 3],
  "test_source_indices": [4, 5, 6]
}
```

索引只放在 JSON 元数据里，不写入 CSV。

```python
    ...

    ...

def dataframe_to_xy(df, label_col="class"):
    ...

def row_to_feature_text(row, feature_categories=None, label_col="class"):
    ...
```

要求：

3. `row_to_feature_text` 只输出值为 `1` 的活跃特征；如果能读到 `dataset-features-categories.csv`，按特征类别分组输出。

## 5. `baseline_fewshot.py`

### 5.1 命令

```bash
python baseline_fewshot.py --split-root data/processed/fewshot_seed42_test100 --k 5
```

可选参数：

```text
--output-root            默认 results/baseline
--seed                   默认 42，仅用于模型随机状态
```

### 5.2 读取数据

读取：

```text
data/processed/fewshot_seed42_test100/k5/train.csv
data/processed/fewshot_seed42_test100/test.csv
```

训练只使用 `train.csv`，评估只使用共用 `test.csv`。

### 5.3 模型

复用当前 notebook 中的 5 个传统 ML 模型：

```text
Logistic Regression
Linear SVM
Random Forest
Decision Tree
KNN
```

注意：KNN 的 `n_neighbors` 不能超过训练样本总数。实现时使用：

```text
n_neighbors = min(5, len(X_train))
如果 n_neighbors 是偶数且大于 1，则减 1
```

这样 `K=1` 时不会报错。

## 6. `llm_only_fewshot.py`

### 6.1 命令

完整运行：

```bash
python llm_only_fewshot.py --split-root data/processed/fewshot_seed42_test100 --k 5
```

调试小样本运行：

```bash
python llm_only_fewshot.py --split-root data/processed/fewshot_seed42_test100 --k 5 --max-test-samples 5
```

可选参数：

```text
--model                  默认 gemma4:e4b
--feature-category-path  默认 data/dataset-features-categories.csv
--output-root            默认 results/predictions
--temperature            默认 0.1
--max-test-samples       默认不限制，仅用于调试
```

### 6.2 Prompt 输入

每次预测一个测试样本时，Prompt 包含：

1. 任务说明：Android 应用恶意/良性二分类。
2. 标签定义：`B = Benign`，`S = Malware`。
3. `train.csv` 中全部 `2K` 个 few-shot examples。
4. 当前测试样本的活跃特征文本。
5. 严格 JSON 输出要求。

Few-shot example 中的标签必须保持 `B` / `S`。

### 6.3 LLM 输出 JSON

要求模型输出：

```json
{
  "pred_label": "S",
  "evidence": ["SEND_SMS", "READ_PHONE_STATE"],
  "explanation": "reason text",
  "confidence": 0.82
}
```

解析后保存 JSONL，每条记录为：

```json
{
  "idx": 0,
  "true_label": "S",
  "pred_label": "S",
  "parse_ok": true,
  "evidence": ["SEND_SMS", "READ_PHONE_STATE"],
  "explanation": "reason text",
  "confidence": 0.82,
  "raw": "original model output"
}
```

如果解析失败：

```json
{
  "idx": 0,
  "true_label": "S",
  "pred_label": null,
  "parse_ok": false,
  "raw": "original model output"
}
```

### 6.4 输出

输出文件：

```text
results/predictions/llm_only_seed42_test100_k5.jsonl
results/predictions/llm_only_seed42_test100_k5_metrics.json
```

评估时需要同时给出：

```text
parse_ok_rate
accuracy_strict        # parse 失败按错误算
accuracy_valid_only    # 只在 parse_ok=true 的样本上算
precision
recall_malware
f1
macro_f1
TP
TN
FP
FN
```

主报告优先使用 `accuracy_strict` 和 `macro_f1`，避免解析失败被忽略后虚高。

## 8. 后续 RAG 衔接约定

本实现不写 RAG 代码，但后续 RAG 必须复用同一数据接口：

```text
train: data/processed/fewshot_seed42_test100/k5/train.csv
test:  data/processed/fewshot_seed42_test100/test.csv
```

这样 LLM-only 和 LLM+RAG 的唯一区别才是是否加入检索规则，而不是数据划分不同。

## 9. 完成标准

本改造完成后，应能形成以下主实验表格：

| 方法 | 训练/示例数据 | 测试数据 | 输出 |
|---|---:|---:|---|
| 传统 ML few-shot baseline | 每类 K 个 | 每类 100 个固定测试集 | `baseline_metrics.csv` |
| LLM-only | 每类 K 个 few-shot examples | 同一个固定测试集 | JSONL + metrics JSON |
| LLM + RAG | 后续复用同一 split | 同一个固定测试集 | 后续实现 |

第一版主实验固定：

```text
K = 5
seed = 42
test_per_class = 100
```
